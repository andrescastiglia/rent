# Arquitectura y Calidad

- Priorizar la información y directrices de la documentación en la carpeta **`docs/`** sobre cualquier otra fuente.
- Si la complejidad ciclomática de una función es **superior a 10**, generar un plan de refactorización para reducirla.
- **Modularidad:** Asegurar que las clases y módulos tengan una única responsabilidad (Single Responsibility Principle - SRP).
- **Cohesión:** Mantener las dependencias internas de los módulos lo más estrechas posible.
